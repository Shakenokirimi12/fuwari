## 学園祭実行委員会が孕んでいた課題

弊学こと、会津大学の学園祭は「蒼翔祭」と呼ばれ、毎年 10 月に開催されます。
蒼翔祭は、学生が主体となって企画・運営を行う学園祭であり、多くの学生が参加し、様々なイベントや展示が行われます。
しかし、昨年までのの蒼翔祭は、以下のような課題を抱えていました。

- **アナログな運営**: 多くの運営作業が手作業で行われており、効率が悪く、ミスも多発していました。
- **情報共有の不足**: 運営メンバー間での情報共有が不十分で、連絡ミスや情報の遅れが頻発していました。
- **効果測定が存在しない**: 学園祭の運営において、業務内容の効果測定が行われておらず、改善点の把握が困難でした。
- **過去の経験が仇になる**: 学園祭を取り巻く環境は年々変化しており、過去の成功体験や失敗体験に良くも悪くも固執することで、新しいアイデアや技術の導入が妨げられていました。

これらの課題を解決するために、私は学園祭実行委員会に**DX 部署**を新設し、デジタル技術を活用して運営の効率化と情報共有の改善を図ることを提案しました。

## DX 部署の設立と活動内容

DX 部署は、「IT 部」と名付けられ、学園祭実行委員会内に以下を目標として新たに設立されました。

- **デジタルツールの導入・運用改善**: 既存の Discord サーバーに AI ツールを導入したり、情報共有ツールを試験運用し、学園祭内の情報共有を(多少)改善しました。
- **データ分析の活用**: 学園祭の運営データ・広報データを大量に収集し、学園祭実行委員会が取り組んだ内容に対する具体的かつ根拠のあるフィードバックを行いました。
- **来場者利便性の向上**: 来場者向けにデジタルマップやイベントスケジュールアプリを開発し、来場者の利便性を向上させました。また、LINE 公式アカウントを運用することで継続的なリーチを実現しました。
- **運営方の利便性の向上**: 備品管理システムの運用を行うことで、学園祭で利用する備品の管理を簡略化し、また、利用した団体やその状況の追跡を行いました。
- **新技術の導入**: 最新のデジタル技術やトレンドを積極的に取り入れ、学園祭の運営に革新をもたらすことで、「会津大学らしい」学園祭を実現しました。

DX 部署の活動は、学園祭実行委員会内でのデジタル化を推進し、運営の効率化と情報共有の改善に寄与することを目的としていました。

##　 IT 部が行った改善と成果・技術スタック

### Trackable Links

学園祭の広報活動において、どの媒体からどれほどのアクセスが来ているかを正確に分析するために行ったのが、**Trackable Links**の開発です。
Hono + Cloudflare Workers + D1 を用いて、宛先 URL と媒体情報を含む短縮 URL を生成し、クリックされた際に元の URL へリダイレクトする前にクリック情報を D1 に保存する仕組みを構築しました。
これにより、各広報媒体の効果を正確に把握できるようになり、次回以降の広報戦略の改善に役立てることができました。

例えば、Instagram の投稿やストーリーズ、Twitter の投稿、ポスターの QR コードなど、様々な媒体にすべて異なる Trackable Link を設定し、どの媒体が最も効果的であったかを分析しました。
また、会津若松市をはじめとした近隣に配布させていただいたポスターでも、すべてのポスターに異なる Trackable Link を設定し、どの地域からのアクセスが多いかも分析しました。
さらに、Trackable Links のアクセス情報の設定画面を、設定がされていない URL に対して表示したり、QR コードリーダーを搭載した LIFF と連携した内部向け LINE 公式アカウントを運用し、効率的に Trackable Links の場所情報の設定ができるようにしました。
これらにより、広報活動の効果測定が大幅に向上しました。
一方で、ポスターから QR コードを読み込むことだけがユーザーの目的ではなく、単にポスターを見るだけでも広報の効果は現れることから、必ずしも Trackable Links のクリック数が広報効果の全てを表しているわけではない、という点がわかりました。
しかし、学園祭における広報活動において、Trackable Links は非常に有用なツールであり、今後も活用していきたいと考えています。

また、本内容を Cloudflare Workers Tech Talks in Niigata #1 でお話したところ、カンファレンス・ハッカソンなどの告知プラットフォームである connpass においてリーチ分析をしたいという希望があったことから、OSS 化も視野に今後も開発を続けていく予定です。

### 備品管理システム

学園祭の運営にあたり、屋外露天や講義棟の中の出店に対して大学で普段使用している机や椅子、また学園祭実行委員会の備品を貸し出す必要があります。
しかし、これらの備品の管理は従来、番号のみが振られたシールを貼り付けた備品を手作業で管理しており、どの団体がどの備品を借りているか、また返却されたかどうかの追跡が困難でした。
そこで、IT 部では備品管理システムを開発し、備品の貸し出し・返却の管理をデジタル化しました。
具体的には、備品に QR コードを貼り付け、貸し出し・返却時に QR コードをスキャンすることで、備品の状態をリアルタイムで把握できるようにしました。
これにより、備品の管理が効率化され、紛失や返却漏れのリスクが大幅に減少しました。
また、備品管理システムは Cloudflare Workers + D1 + Hono + React + TailwindCSS を用いて開発され、使いやすい UI を提供しました。
さらに、備品の貸し出し状況をリアルタイムで確認できるダッシュボード(後述の Sho-Room に実装)も用意し、運営メンバーが迅速に対応できるようにしました。

### Sho-Room

IT 部が作成したシステムのすべてを統合的に管理・運用するためのダッシュボードとして、**Sho-Room**を開発しました。
Sho-Room は、学園祭実行委員会の運営メンバーが利用することを目的としており、以下の機能を提供しました。

- **備品管理ダッシュボード**: 備品管理システムと連携し、貸し出し状況や返却状況をリアルタイムで確認できる機能を提供しました。
- **Trackable Links 管理**: Trackable Links の生成・管理機能を提供し、各広報媒体の効果測定を容易にしました。
- **来場者データ分析**: 来場者の行動データを分析し、学園祭の運営改善に役立てるためのデータ可視化機能を提供しました。
- **ユーザー管理**: 運営メンバーのアカウント管理機能を提供し、権限設定やアクセス制御を行えるようにしました。
- **Discord ログイン連携**: 学園祭実行委員会が利用している Discord サーバーと連携し、運営メンバーが Sho-Room に簡単にログインできるようにしました。また、ロール情報から利用者の権限を自動的に設定する仕組みも導入しました。

Sho-Room は、Cloudflare Workers + D1 + Hono + React + TailwindCSS を用いて開発され、使いやすい UI と高いパフォーマンスを実現するよう実装を行いました。
これにより、学園祭実行委員会の運営メンバーは、IT 部が提供する各種システムを一元的に管理・運用できるようになりました。
一方、Sho-Room 自体の利用頻度はそれほど高くなく、IT 部が提供する各種システムの利用頻度も限定的であったため、今後はより多くの運営メンバーに利用してもらえるような仕組みを検討していく必要があると感じました。

### 広報用 Web サイト

学園祭の広報活動を支援するために、IT 部では広報用 Web サイトを開発しました。
例年は広報部 Web 課が担っていた内容を IT 部に移管した形です。
広報用 Web サイトは、学園祭の情報を効果的に発信することを目的としており、以下の機能を提供しました。

- **イベント情報の掲載**: 学園祭で開催される各種イベントの情報を掲載し、来場予定者に分かりやすく提供しました。
- **アクセス情報の提供**: 学園祭会場へのアクセス方法や交通手段を詳しく説明し、来場者の利便性を向上させました。
- **FAQ セクション**: よくある質問とその回答を掲載し、来場者の疑問を解消しました。

まぁ、一般的な Web サイトですね。
広報用 Web サイトは、Cloudflare Pages + Next.js + TailwindCSS を用いて開発され、モバイルフレンドリーなデザインと高速なパフォーマンスを実現しました。
これにより、学園祭の広報活動が強化され、来場者の関心を引き付けることができました。

### LINE 公式アカウント

後述する来場者向けアプリを使用する元として、学園祭の LINE 公式アカウントを開設・運用しました。
本アカウントでは、

- **こちらからの情報告知**: 学園祭の最新情報やイベント情報を定期的に配信しました。
- **来場者からの問い合わせ対応**: よくあるご質問をご用意し、自動応答で対応しました。

を行いました。
企画部が管轄するプロジェクト「あかべこがーる」では、企画があることを来場者に周知するために、LINE 公式アカウントを活用しました。
告知の際、アイコンが「あかべこがーる」になるなど、来場者に親しみやすい工夫も行いました。
Hono + Cloudflare Workers + D1 を用いて、LINE Messaging API と連携し、効率的な情報発信と問い合わせ対応を実現しました。

### 来場者向けアプリ

来場者アプリは公式アカウントに紐づけられた LIFF アプリとして実装されており、来場者は LINE アプリ内でシームレスに利用できるようになっていました。
来場者アプリでは、以下の機能を提供しました。

- **来場者 QR の表示**: 来場者が学園祭に来場した際に提示するための QR コードを表示しました。これにより、来場者の入場管理が効率化されました。
- **デジタルマップ**: 学園祭会場のデジタルマップを提供し、来場者が目的の場所を簡単に見つけられるようにしました。
- **イベントスケジュール**: 学園祭で開催される各種イベントのスケジュールを掲載し、来場者が参加したいイベントを把握できるようにしました。
- **イベント予約**: 学園祭で開催されるイベントのうち予約が可能なイベントに関しては、来場者から予約を行い優先的に案内ができるようにしました。

## 結局何がやりたかったのか？それはできたのか？

IT 部を設立した目的は、学園祭実行委員会の運営をデジタル化し、効率化と情報共有の改善を図ることでした。
結果的に、いくつかのシステムの導入により、特に「数値的な効果測定を行う」という点では大きな成果を上げることができました。
来場者の皆様にお願いしたアンケートでは、「会津大学らしい学園祭だ」と肯定的な意見を多くいただくことができ、IT 部の取り組みが一定の評価を受けたと感じています。
学園祭実行委員会の中での IT 部に対する評価も概ね良好であり、IT 部の活動が学園祭の運営に貢献したと考えています。
しかし、一方で、IT 部が提供するシステムの利用頻度が限定的であったり、運営メンバー全体への浸透が不十分であったりと、課題も残りました。
